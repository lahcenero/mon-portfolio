Remise via GitHub Classroom

Les comptes rendus des travaux pratiques doivent être rendus via GitHub Classroom. Cette plateforme permet de gérer les dépôts Git pour chaque exercice et facilite la correction et le suivi.

🔗 Accéder à GitHub Classroom

Comment se connecter et lier votre compte GitHub

Pour accéder aux exercices et rendre vos travaux, vous devez d'abord vous connecter à GitHub Classroom et lier votre compte GitHub.

  1. Créer un compte GitHub : si vous n'avez pas encore de compte GitHub, créez-en un sur github.com (gratuit).
  2. Accéder à GitHub Classroom : cliquez sur le lien “Accéder à GitHub Classroom”.
  3. Se connecter avec GitHub : cliquez sur “Sign in to GitHub” et autorisez GitHub Classroom à accéder à votre compte.
  4. Lier votre compte dans le Classroom Roster :
    • Une fois connecté, vous verrez la liste des étudiants (Classroom Roster)
    • Recherchez votre nom dans la liste
    • Cliquez sur votre nom pour le sélectionner
    • Confirmez que c'est bien votre compte GitHub qui est associé
  5. Accepter les exercices : une fois votre compte lié, vous pourrez accepter les assignments. Chaque exercice créera automatiquement un dépôt Git privé dans votre compte GitHub.

Rendre votre travail

Pour chaque exercice :

  1. Un dépôt Git privé est créé automatiquement dans votre compte GitHub
  2. Clonez ce dépôt sur votre machine locale
  3. Ajoutez vos fichiers (code, compte rendu, README, etc.)
  4. Commitez et poussez vos modifications vers GitHub
  5. Le dépôt sera automatiquement visible par l'enseignant pour correction
Note importante : assurez-vous de bien lier votre compte GitHub à votre nom dans le Classroom Roster avant de commencer. Si vous rencontrez des difficultés, contactez l'enseignant.
